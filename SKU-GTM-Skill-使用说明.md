# SKU GTM资产产出 Skill 使用说明

适用Skill文件：
- `/Users/cressvanish/Documents/New project/skills/asset-factory-v1/SKU GTM资产产出SKILL.md`
- 系统识别入口：`/Users/cressvanish/Documents/New project/skills/asset-factory-v1/SKILL.md`

## 1. 你每次只需要给我这些输入
1. Brief（可用模板：`/Users/cressvanish/Documents/New project/ops/SKU-GTM-BRIEF-template.md`）
2. 目标平台
3. 资产范围（KV/主图/详情/视频/是否含产品单页和手册中英版）
4. 时间要求（默认72小时）

## 2. 标准调用话术（你直接复制发我）
`按 SKU GTM资产产出SKILL 跑这个SKU。以下是Brief与平台要求，请输出完整交付包。`

## 3. 我会一次性返回的交付包
1. Brief预检报告（完整性+冲突）
2. 平台规范矩阵（锁版建议）
3. AI资产执行包（KV/主图/详情/视频关键帧）
4. Challenge-ready资产决策说明卡（每个关键资产1张）
5. 三重验收评分表（AI/Human/Ops）
6. 放行结论（资产级+SKU级）
7. 若不通过：48小时整改任务单

## 4. 团队执行顺序
1. GTM-PM确认Brief并锁版
2. 执行组按AI执行包生产素材
3. AI QA先评
4. Human QA复评
5. Ops QA终评
6. GTM-PM裁决签发

## 5. 判定门槛
- 资产级：AI>=85，Human通过，Ops>=80，且无红线
- SKU级：KV/主图/详情全通过才可签发

## 6. 常见失败点（先避坑）
1. 主图与详情首屏叙事不一致
2. 卖点没有证据锚点
3. 描述过强缺边界说明，导致“商品与描述不符”风险
4. 权益位在不同素材里漂移，影响Ops投放承接

## 7. 版本更新规则
- 这个Skill可持续更新，不是一次性定稿。
- 每双周复盘后可修改：
  - 评分权重
  - 红线规则
  - Prompt模板
  - 挑战应答口径
